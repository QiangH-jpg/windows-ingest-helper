# 元泉智影上传助手 — Mac 实机测试说明

## 1. 下载 App

1. 打开 GitHub 仓库：https://github.com/QiangH-jpg/windows-ingest-helper
2. 点击 **Actions** 标签
3. 找到最新的 **Build macOS Uploader** workflow run
4. 点击进入 → 页面底部 **Artifacts** 区域
5. 下载 `openclaw-uploader-macos-arm64`
6. 解压 ZIP 文件

## 2. 打开未签名 App

macOS 会阻止未签名 App 运行。按以下步骤处理：

### 方式 A：右键打开（推荐）

1. 找到 `元泉智影上传助手.app`
2. **右键** → **打开**
3. 弹出"无法验证开发者"对话框 → 点击 **打开**
4. 后续可正常双击打开

### 方式 B：系统设置

1. 尝试双击打开 App（会被阻止）
2. 打开 **系统设置** → **隐私与安全性**
3. 向下滚动，找到"已阻止…"提示
4. 点击 **仍要打开**

### 方式 C：命令行（如出现"已损坏"）

```bash
# 清除隔离标记
xattr -cr "元泉智影上传助手.app"

# 打开
open "元泉智影上传助手.app"
```

## 3. 测试步骤

### Step 1：连接服务器

1. 打开 App
2. 服务器地址默认为 `http://47.93.194.154:8088`
3. 点击 **测试** 按钮
4. 预期：显示 ✅ 已连接

### Step 2：检测 FFmpeg

1. 点击 **检测** 按钮
2. 预期：显示 ✅ ffmpeg 和 ✅ ffprobe

### Step 3：输入主题/事件

1. 填写"视频主题"（如：测试上传）
2. 填写"新闻事件"（如：macOS 客户端首次测试）

### Step 4：选择素材文件夹

1. 点击 **选择文件夹**
2. 选择包含 mp4/mov 视频的文件夹
3. 预期：显示"发现 N 个视频文件"

### Step 5：开始转码上传

1. 点击 **开始转码上传**
2. 等待串行处理（每个文件：转码 → presign → PUT）
3. 预期：每个文件显示 ✅ uploaded
4. 最后显示 notify 状态 = processing

### Step 6：打开 Web 工作台

1. 上传成功后自动打开浏览器
2. 或点击 **打开 Web 工作台** 按钮
3. 在 Web 工作台中可以看到刚上传的 task

## 4. 预期结果

- 正常视频（>1.5s，>100KB）：转码 → 上传 → 成功
- 短片（≤1.5s）：跳过
- 坏片（<360p / <100KB）：跳过
- 上传完成后在 Web 工作台可见
- 后续生成/返修在 Web 工作台进行

## 5. 已知限制

| 限制 | 说明 |
|------|------|
| 未签名 | 首次需右键打开或 xattr 清除 |
| 仅 Apple Silicon | 不支持 Intel Mac |
| 无断点续传 | 上传中断需重新开始 |
| 无并发上传 | 串行处理，逐个转码上传 |
| 无本地剪辑 | 剪辑/返修在 Web 工作台 |
| 无本地 AI | AI 处理在服务器 |
| 无上传历史 | 不保存历史记录 |
| 无进度百分比 | 只显示当前文件状态 |
