## 故障记录：视频下载链接 404 问题

### 问题现象
- 新生成的视频样片无法通过 Web 界面下载
- 访问 `/api/download/<task_id>` 返回 404
- 视频文件实际已生成在 `outputs/` 目录

### 问题原因
任务 JSON 文件未注册到正确的任务目录：
- 错误位置：`pipeline/tasks/`
- 正确位置：`workdir/tasks/`

Web 服务从 `workdir/tasks/` 读取任务列表，新任务 JSON 放在错误目录导致无法识别。

### 解决方案
1. **临时修复**：手动复制任务 JSON 到正确目录
   ```bash
   cp pipeline/tasks/<task_id>.json workdir/tasks/
   ```

2. **重启 Web 服务**：
   ```bash
   pkill -f "python.*run.py"
   cd /home/admin/.openclaw/workspace/video-tool
   nohup /home/admin/.openclaw/workspace/.venv/bin/python run.py > logs/web.log 2>&1 &
   ```

3. **验证**：
   ```bash
   curl -s "http://47.93.194.154:8088/api/download/<task_id>" -o /dev/null -w "%{http_code}"
   # 应返回 200
   ```

### 长期修复建议
1. 修改 `run_semantic_v15_full.py` 脚本，将任务 JSON 直接写入 `workdir/tasks/`
2. 或在 `app/main.py` 中添加任务注册逻辑，生成视频后自动注册
3. 或修改 `pipeline/tasks.py` 的 `TASKS_DIR` 配置指向正确路径

### 影响范围
- 语义选片 V1.5 及后续使用独立生成脚本的任务
- 不影响主流程 `run_news_schedule_v2.py`（该脚本已正确注册任务）
