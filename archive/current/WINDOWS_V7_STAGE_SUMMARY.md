## 二、从 v3 到 v7 的关键问题演变

| 版本 | 关键问题 | 解决状态 |
|------|---------|---------|
| v3 | 首次构建成功 | ✅ 解决 |
| v4 | 调试增强但未真正修复 | ⚠️ 未解决 |
| v5 | Chocolatey shim 假 ffmpeg（384 KB） | ❌ 作废 |
| v6 | 真 ffmpeg/ffprobe 带入（96 MB） | ✅ 解决 |
| v7 | ffprobe JSON 解析逻辑失败 | ✅ 解决 |

**v5 作废原因**:
- `bin/ffmpeg.exe` 和 `bin/ffprobe.exe` 各只有 384 KB
- 这是 Chocolatey shim/包装器，不是真正的 ffmpeg 二进制
- 真正的 ffmpeg.exe 应该在 90-100 MB

**v7 修复内容**:
- 明确使用 `stdout=subprocess.PIPE` 和 `stderr=subprocess.PIPE`
- 添加 stdout 是否为空的检查
- 添加 JSON 解析异常处理 (`json.JSONDecodeError`)
- 添加详细调试输出（命令/返回码/stdout/stderr/堆栈）
