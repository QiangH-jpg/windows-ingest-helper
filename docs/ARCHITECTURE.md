# 视频项目架构文档

## 分层架构

```
┌─────────────────────────────────────────────────────────┐
│                  scripts/run_production.py              │
│                    唯一主链入口                          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│  V1 素材层 (v1_materials/)                              │
│  职责：素材管理、转码、缓存                              │
│  输入：uploads/*.mp4                                    │
│  输出：cache/standardized/*.mp4, data/materials.json   │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│  V2 语义层 (v2_semantic/)                               │
│  职责：稿件切分、标签匹配、选片计划                      │
│  输入：data/script.txt                                  │
│  输出：v2_semantic/shot_plan.json                       │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│  V3 时序层 (v3_timeline/)                               │
│  职责：镜头顺序、时长分配、节奏分布                      │
│  输入：shot_plan.json, audio_duration                   │
│  输出：v3_timeline/timeline.json                        │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│  V4 渲染层 (v4_render/)                                 │
│  职责：TTS、字幕、合成                                   │
│  输入：timeline.json, script.txt                        │
│  输出：v4_render/candidate.mp4                          │
└─────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────┐
│  V5 校验层 (v5_gate/)                                   │
│  职责：门禁式校验、发布                                  │
│  输入：candidate.mp4, timeline.json, script.txt        │
│  输出：outputs/approved/*.mp4 或 data/rejection_report.json │
└─────────────────────────────────────────────────────────┘
```

## 唯一入口/唯一输出原则

### 唯一入口

- **生产链入口**: `scripts/run_production.py`
- **Web 服务入口**: `app/main.py`

### 唯一输出

| 层 | 唯一输出 |
|----|----------|
| V1 | `cache/standardized/*.mp4`, `data/materials.json` |
| V2 | `v2_semantic/shot_plan.json` |
| V3 | `v3_timeline/timeline.json` |
| V4 | `v4_render/candidate.mp4` |
| V5 | `outputs/approved/*.mp4` (通过) 或 `data/rejection_report.json` (失败) |

## 禁止事项

1. **禁止跨层直连**: 下一层只能读取上一层唯一输出
2. **禁止绕层**: 不得跳过任何一层
3. **禁止旧链出片**: `legacy/` 目录脚本不得生成合格成片
4. **禁止硬编码路径**: 所有路径通过 `config/settings.py` 配置

## 门禁规则（V5）

以下任一失败则不输出合格成片：

| 规则 | 阈值 | 说明 |
|------|------|------|
| 视频时长 | ≥ 音频 + 0.1s | 防止语音被截断 |
| 最短镜头 | ≥ 1.5s | 防止闪屏 |
| 字幕匹配率 | ≥ 95% | 防止漏词漏句 |
| 无异常片段 | 是 | 防止画面异常 |

## 配置化

所有路径通过环境变量配置：

```bash
# .env 文件
APP_ROOT=/path/to/video_tool
UPLOADS_DIR=${APP_ROOT}/uploads
OUTPUT_DIR=${APP_ROOT}/outputs
...
```

## 可迁移性

1. **代码与配置分离**: 代码不写死路径
2. **缓存可重建**: `cache/` 目录可安全删除
3. **一键初始化**: `python scripts/init_directories.py`
4. **依赖明确**: `requirements.txt` 列出所有依赖
