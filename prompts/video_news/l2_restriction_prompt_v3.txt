## 你只负责一件事：标注"哪些秒不能用"，留下"哪些秒可以选"

### 一、画面质量底线（标注为 unsafe）

逐秒检查，以下任何一项出现即标注 unsafe：
- 明显抖动、手持晃动影响主体识别
- 虚焦、跑焦
- 过曝/欠曝导致信息丢失
- 镜头/手指/设备遮挡主体
- 严重歪斜、噪点严重、水印/异物明显
- 花絮感镜头（拍摄者闲聊、调参、走路等非叙事画面）

### 二、正式度底线（标注为 unsafe）

以下任何一项出现即标注 unsafe：
- 嬉笑/捂嘴笑/打闹/不得体表情
- 搞怪/夸张肢体语言
- 严重失态动作

### 三、边界脏段（标注为 boundary）

以下区间不属于严重问题，但不适合作为正式新闻镜头的起止点：
- **preparation**：人物刚进入画面、找位置、调整站姿、等待指示
- **tail**：转身离开、低头整理、视线游离、收拾物品
- **transition**：起身、走位、交头接耳、摆手、打瞌睡、玩手机、整理衣服
- **camera_shake**：镜头启动晃动（前摇/对焦过程）、镜头收尾拖动（拖尾/收镜晃动）
- **weakly_informal**：虽不严重失态但明显不够正式的段落（动作未完成、表情未到位、有人未就位、散漫状态）

### 四、人物与构图底线（标注为 unsafe 或 boundary）

- 切头/切脸/半脸/主体面部遮挡 → unsafe
- 关键动作被遮挡 → unsafe
- 合影站位混乱/有人未就位 → boundary(weakly_informal)
- 领导/主体形象不端正 → boundary(transition) 或 unsafe

### 五、信息类型判断（单选）

theme / overview / core_service / policy / feedback / detail / non_core_activity / closing

## 硬约束

1. clean_windows 必须同时避开 unsafe_segments 和 boundary_segments
2. clean_windows 是"画面干净 + 正式得体 + 主体完整 + 无过渡动作"的窗口
3. 如果整条素材无 clean_window，usable 必须为 false
4. 所有时间点精确到 0.5 秒
5. **你不需要判断"哪个瞬间最好"，只需要判断"哪些范围可以选"**

只返回 JSON，不要其他内容。
