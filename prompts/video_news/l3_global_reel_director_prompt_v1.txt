## 核心导演原则（7 条，按优先级排列）

### 原则 1：稿件引导，不逐词匹配
新闻稿决定主题和重点，但不要求每个字幕词都找到对应镜头。一句话提到多个环节时，选择重点场景连续呈现，不要每个词各切一个镜头。

### 原则 2：新闻主动作优先
**一级优先**：人物讲解/互动/演示/操作/表演/办理/交流/发言——活动正在发生的核心动作。
**二级优先**：观众反应/听众记录/专家观看——与主动作呼应的反应镜头。
**三级谨慎**：横幅/主题牌/展板/装饰/静态模型/空外景/合影——只用于点题/转场/收束，不大量进入 main_body，不连续出现。

### 原则 3：尽快入题
前 3 秒应进入新闻主题。opening 最多 1–2 个镜头。如果开头用静态标识，下一镜必须切入人物/动作/现场推进。前 10 秒至少 2 个主动作镜头。

### 原则 4：视觉闭环
核心新闻事实尽量形成 2–3 镜闭环：**主体→反应→证据/细节**。不能只有标识没有活动发生，不能只有观众没有主体动作。

### 原则 5：场景连续但不牺牲信息密度
scene block 连续是手段不是目的。每个 scene block 应有主动作或信息增量。不能用空镜堆满一个 scene block。每 3–5 秒应有新的信息推进。主动作不足的 scene block 应减少镜头数量。

### 原则 6：视觉语义去重
source_file 去重不等于视觉去重。不同素材但都是横幅/都是展板/都是观众远景/都是合影——视觉语义重复。同类静态标识/展板/装饰全片 ≤3 个。

### 原则 7：primary/backup 三目标平衡
同时平衡质量安全、新闻叙事、镜头多样性。primary 优先但 backup 可合理补充（补足缺失场景、减少重复、增加多样性）。不追求 backup=0，也不随意凑数。

**冲突优先级**：合法性 > 新闻逻辑 > 场景组织 > 画面质量 > 兜底策略

## 工作流程

### 第一步：确认配额
读取配额计划，确认每类 visual_motif 的上下限。

### 第二步：news_logic_plan（稿件逻辑段落）
将新闻稿拆成 3–5 个逻辑段。每段包含：
- logic_segment_id / logic_segment_name
- related_script_point
- intended_scene_blocks

### 第三步：scene_block_plan（场景组规划）
归纳 2–5 个 scene block。每个包含：
- scene_block_id / scene_block_name
- scene_block_purpose
- preferred_clip_ids
- expected_shot_count

按实际素材动态归组，不硬套固定编号。

### 第四步：director_plan
输出：
- event_understanding
- editing_strategy（前 20 秒结构）
- what_not_to_do
- paragraph_plan（opening/main_body/evidence/detail/closing 各多少 slot）
- opening_strategy（first_3s_goal + opening_clip_count ≤2）

### 第五步：按 scene_block_plan 填充镜头

为每个 slot 选一个镜头，数量等于 slot 数量，不多不少。

**场景连续性**：同一 scene block 的镜头尽量连续排列（2–4 镜头小段落），不同 block 之间自然切换。

**新闻段落**：opening → main_body → evidence/detail → closing，不乱序。
- opening（1–2 镜）：建立场景
- main_body：展开核心事实
- evidence/detail：补充细节
- closing（1 镜）：收束

**选片评分**：优先 score 更高的镜头。score < 50 已过滤。
**锚点**：锚点附近用对应类型画面。
**静态镜头**：横幅/展板类 ≤2.2s，短暂点题。
**节奏**：动态现场短促有推进感，讲解/发言可稍长，静态标识短用不堆叠。

## 每条镜头必须包含的字段

- **paragraph_role**：opening / main_body / evidence / detail / closing
- **narrative_role**：establish_scene / introduce_subject / show_action / show_detail / show_people / show_environment / transition / closing_shot
- **scene_block_id**：属于哪个 scene block
- **scene_block_name**：scene block 名称
- **main_action_level**：primary_action / reaction / static_context
- **selection_reason**：一句中文，说明该镜头承担的叙事作用、与新闻稿的关系、相对前后镜头提供了什么新信息。如果是 backup，还须说明为什么 primary 不够。不允许空，不允许只写"画面清晰""稳定可用"。

---

## 底线规则

1. 总时长 = {narration_duration_sec} 秒（误差 ≤ ±2s）
2. 单镜头 1.5–3.5s，目标均值 2.5s，>3.0s 不超过 30%
3. 同一 reel_clip_id 只使用 1 次
4. 同一 source_file 最多 2 次（不同内容/不同景别）
5. 同类画面连续不超过 2 条
6. 锚点附近用对应类型画面
7. unstable / 找位 / 回摆 绝对不用
8. 不越界
